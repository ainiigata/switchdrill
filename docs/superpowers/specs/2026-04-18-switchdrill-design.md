# SwitchDrill — 設計ドキュメント

**作成日:** 2026-04-18  
**概要:** 注意切り替え力（アテンション・スイッチング）を鍛えるゲーミフィケーションWebアプリ。ビジネスパーソン向け。

---

## 1. コンセプト

AIの時代において「複数の文脈を素早く切り替えながら作業する力」は必須スキル。SwitchDrillは異なる種類のミニゲームをランダムな順番で次々プレイさせることで、注意の切り替え速度と正確さを科学的に鍛える。

---

## 2. コアゲームループ

```
セッション開始
  ↓
ランダムミニゲーム表示（3〜7秒の制限時間）
  ↓
クリア/タイムアウト → 次のミニゲームへ即座に切り替え
  ↓
10問でセッション終了 → スコア・ランク表示
  ↓
レベルアップ判定 → 次セッションの難易度調整
```

---

## 3. ミニゲーム（5種類）

### 3-1. 計算スイッチ
- 簡単な算数問題（例: 7 × 8 = ?）を素早く正解
- 難易度: 数字の桁数・演算子の種類で調整

### 3-2. ストループテスト
- 「赤」という文字が青色で表示 → 「文字の色」を選ぶ
- 認知科学的に注意切り替え力と直結する古典的テスト
- 難易度: 選択肢の数で調整

### 3-3. パターン記憶
- 3×3〜5×5グリッドのパターンを0.8秒表示 → 再現
- 難易度: グリッドサイズ・表示時間で調整

### 3-4. カテゴリ分類
- 表示された単語を「仕事/プライベート」「動物/食べ物」などに振り分け
- 難易度: カテゴリ数・紛らわしい単語で調整

### 3-5. 反応タップ
- 特定の図形/色が出たらタップ、違う場合はスルー
- ゴーノーゴーテスト（抑制制御）
- 難易度: 出現速度・紛らわしい図形で調整

---

## 4. スコアシステム

```
問題スコア = 基準点(100) × 速度係数 × 正確性
速度係数 = max(0.2, 1 - 経過時間/制限時間)
```

- **コンボボーナス:** 連続正解で×1.1〜×2.0
- **切り替えボーナス:** 前の問題と異なる種類のゲームへの切り替えで+10pt
- **セッションランク:** S(95%+) / A(80%+) / B(65%+) / C(50%+) / D(それ以下)
- **レベル:** 1〜50。セッション完了で経験値獲得。Lvが上がると難易度・速度が上昇。

---

## 5. 進捗・モチベーション設計

- **ダッシュボード:** 過去セッションの成績グラフ（折れ線）、ゲーム種別の得意/苦手レーダーチャート
- **ストリーク:** 連続プレイ日数カウント、バッジ表示
- **実績システム:** 「ストループ達人」「10コンボ」などのアンロック実績
- **所要時間:** 1セッション約2〜3分（ビジネスパーソンが隙間時間に使える）

---

## 6. アーキテクチャ

### 技術スタック
- **フレームワーク:** React 18 + TypeScript
- **ビルド:** Vite
- **スタイル:** Tailwind CSS
- **状態管理:** Zustand（軽量、シンプル）
- **データ永続化:** localStorage（サーバー不要、無料）
- **グラフ:** Recharts

### ディレクトリ構成
```
src/
  components/
    games/          # 各ミニゲームコンポーネント
      CalcGame.tsx
      StroopGame.tsx
      PatternGame.tsx
      CategoryGame.tsx
      ReactionGame.tsx
    ui/             # 共通UIコンポーネント
      TimerBar.tsx
      ScoreDisplay.tsx
      ComboIndicator.tsx
  pages/
    Home.tsx        # トップ/スタート画面
    Game.tsx        # ゲームセッション画面
    Result.tsx      # セッション結果画面
    Dashboard.tsx   # 進捗ダッシュボード
  store/
    gameStore.ts    # Zustand store（セッション状態）
    progressStore.ts # Zustand store（進捗・レベル）
  hooks/
    useGameSession.ts  # ゲームセッションロジック
    useTimer.ts        # タイマーロジック
  types/
    index.ts        # 型定義
  utils/
    scoring.ts      # スコア計算ロジック
    gameSelector.ts # ゲームランダム選択ロジック
    difficulty.ts   # 難易度パラメータ生成
```

### データフロー
```
progressStore (レベル・実績・履歴)
    ↓ 難易度パラメータ
gameStore (セッション状態)
    ↓ 問題・タイマー
Game.tsx → 各ミニゲームComponent
    ↓ 回答イベント
useGameSession → スコア計算 → gameStore更新
    ↓ セッション完了
Result.tsx → progressStore更新 → ダッシュボードへ
```

## 7. 画面設計

### ホーム画面
- アプリ名・キャッチコピー
- 現在のレベル・ストリーク表示
- 「スタート」ボタン（大きく中央）
- ダッシュボードへのナビゲーション

### ゲーム画面
- 上部: タイマーバー（残り時間を視覚的に表示）
- 中央: ミニゲームエリア（全画面の60%）
- 上右: 現在スコア・コンボ表示
- 下部: ゲーム種類インジケーター（何問目/10問）

### 結果画面
- ランク表示（大きなA/S/Bなど）
- 獲得スコア・平均反応速度
- ゲーム種別の正解率内訳
- 解放された実績（あれば）
- 「もう一度」「ダッシュボードへ」ボタン

### ダッシュボード
- 過去14日の成績グラフ
- レベル・EXPバー
- 得意/苦手ゲームのレーダーチャート
- 実績バッジ一覧
